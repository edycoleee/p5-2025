class Ball {
  constructor(x, y, r) {
    this.x = x;
    this.y = y;
    this.r = r;
  }

  // Method untuk menampilkan bola

    this.y += random(-2, 2);
  }
}

  bola.move();     // bola bergerak
  bola.display();  // bola ditampilkan
}

ğŸ¯ Ringkasan
Class = blueprint (Kotak, Bola)

Object = hasil nyata (kotak1, bola1, dll.)

Method/Function = aksi (moveRight, move, display)

Struktur yang kamu buat itu terdiri dari:

Class Tree

Blueprint untuk pohon biner.

Punya properti root (akar pohon).

Punya method AddNode() â†’ aksi untuk menambahkan node ke pohon.

Class Node

Blueprint untuk simpul pohon.

Punya properti value (nilai yang disimpan), left (anak kiri), dan right (anak kanan).

ğŸ‘‰ Jadi benar: AddNode adalah method milik Tree, karena logika menambah node ke pohon memang tanggung jawab class Tree. Sedangkan class Node hanya mendefinisikan bentuk simpulnya saja.

Tree = wadah besar

AddNode() = cara menaruh simpul ke dalam pohon

Kalau mau lebih jelas, bisa kita kembangkan contoh tadi:

javascript
var tree;

function setup(){
    tree = new Tree();
    tree.AddNode(new Node(5));   // root
    tree.AddNode(new Node(3));   // anak kiri
    tree.AddNode(new Node(7));   // anak kanan
    console.log(tree);
}

function Tree(){
    this.root = null;
}

Tree.prototype.AddNode = function(n){
    if (this.root == null){
        this.root = n;
    } else {
        this._AddNode(this.root, n);
    }
}

Tree.prototype._AddNode = function(current, n){
    if (n.value < current.value){
        if (current.left == null){
            current.left = n;
        } else {
            this._AddNode(current.left, n);
        }
    } else {
        if (current.right == null){
            current.right = n;
        } else {
            this._AddNode(current.right, n);
        }
    }
}

function Node(val){
    this.value = val;
    this.left = null;
    this.right = null;
}

ğŸ“˜ Konsep Search di Binary Tree
Mulai dari root.

Bandingkan nilai yang dicari dengan nilai node saat ini:

Jika sama â†’ ketemu.

Jika lebih kecil â†’ lanjut ke anak kiri.

Jika lebih besar â†’ lanjut ke anak kanan.

Jika sampai ke node null â†’ berarti nilai tidak ada di pohon.

ğŸ–¥ï¸ Implementasi Search
Tambahkan method search(val) ke dalam class Tree:

javascript
Tree.prototype.search = function(val){
    return this._search(this.root, val);
}

Tree.prototype._search = function(current, val){
    if (current == null){
        return null; // tidak ketemu
    }

    if (val == current.value){
        return current; // ketemu
    } else if (val < current.value){
        return this._search(current.left, val); // cari ke kiri
    } else {
        return this._search(current.right, val); // cari ke kanan
    }
}

ğŸ¯ Ringkasan
Tree = class utama, punya root.

Node = simpul dengan value, left, right.

AddNode() = method untuk menambah node sesuai aturan kiri/kanan.

search(val) = method untuk mencari node dengan nilai tertentu.

ğŸ“˜ Kenapa ada 2 class (Tree dan Node)?
Tree = wadah besar, hanya ada satu objek utama (tree) yang menyimpan keseluruhan struktur.

Node = simpul pohon, jumlahnya bisa bertambah banyak sesuai data yang kita masukkan.

Jadi:

Kita definisikan class Tree sekali, karena hanya ada satu blueprint pohon.

Kita definisikan class Node sekali, karena setiap kali kita ingin menambah data, kita buat object baru dari blueprint Node.

ğŸ—ï¸ Bagaimana cara kerjanya?
Tree punya properti root.

Awalnya null.

Saat kita tambahkan node pertama, dia jadi root.

Node adalah blueprint simpul.

Setiap kali kita new Node(val), kita bikin simpul baru dengan nilai val.

Node ini bisa ditempatkan di kiri atau kanan sesuai aturan binary tree.

Semua node yang dibuat disimpan di dalam Tree, karena Tree adalah variabel utama yang mengorganisir mereka.

ğŸ” Analogi sehari-hari

Node = laci di dalam lemari.

Kita hanya punya satu lemari utama (Tree), tapi bisa menambah banyak laci (Node).

Method AddNode() = cara memasukkan laci baru ke posisi yang tepat (kiri/kanan).
